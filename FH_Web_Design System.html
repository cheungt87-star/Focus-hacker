00 · Identity

A calm warmth, with sharp teeth.

FocusHacker is built on a small contradiction: the soft cream-and-lavender warmth of a journaling app, paired with the kernel-level firmness of a real blocker. The visual system carries that same tension.

01

Warm, not soft.

Cream backgrounds, serif headlines, hand-placed stickers. The page should feel like the inside of a good notebook — never sterile, never gamer-y.

02

Confident, not loud.

Lavender carries the brand. Saturated colors live inside the app screenshots. The marketing surface never shouts; it lets the product do the shouting.

03

Italic emphasis.

Real italics inside serif headlines — paired with a lavender chip behind the key phrase — are the brand's signature move. Use them sparingly.

04

Stickers, not stickers.

The floating "+25 XP" pills aren't decoration — they're how we surface gamification without putting it in the chrome. They lean. They overlap. They're a voice.

06 · Tokens

Drop it into any project.

Everything above resolves to a single tokens.css file. Link it once, then reach for the named variables everywhere else.

Step 1

Link the file.

<link rel="stylesheet" href="tokens.css">
Step 2

Reach for the vars.

.hero {
  background: var(--surface-page);
  color: var(--ink);
  font-family: var(--font-display);
}
.btn-primary {
  background: var(--lavender);
  border: 2px solid var(--ink);
  border-radius: var(--radius-md);
  box-shadow: var(--shadow-glow-lavender);
}
Step 3

Stay inside the system.

No raw hex codes outside the screenshot palette. No font names outside --font-display / --font-ui / --font-mono. No spacing values that aren't on the 4px grid.
